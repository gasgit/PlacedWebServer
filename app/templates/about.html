Abstract


The objective for this report is to document the design, development of a software application called Placed. Placed is an application where users will create a collection of places and things of interest onto a server that can be viewed or created using an Android device. All users of the application can access the collection of posts made while visiting an area, view comments made, when the post was created, an image, view a map, view current location, view other posted locations and save favourites on to the device. Users will be able to select a geofence of the area to request a filtered list of places to the device ie. 1km, 5km and 10km.













The project will remain as prototype and in a local development environment until submission with some testing on actual devices.

2.4 System Architecture and Design


I have decided to create the application with a MVC framework. Separating UI (View) from Database(Model) and Web Server(Controller). The mobile applications requires user interaction(view) to retrieve/send data to the model(database) . This database is accessed by all users and needs to be stored on a server. All communication and data manipulation needs to go through a web service(controller)

2.4.1 Model View Controller


Model is the database where the data is stored with the rules and constraints.The model will return the same set of data for a controller request whether the request comes from the mobile app or a webpage on the server. The model will place the data into a data structure the controller can use to return the data to the view. In this system the controller will jsonify


View is the presentation lay of the application, designed as to how the data is presented to the user on the device. On mobile application in this system it will be an activity with a view and on the web server it will be a HTML page.


Controller manages the requests from the user interface and manipulates the data from the model and returns requests back to the view.

2.4.2 HTTP


HTTP is the most commonly used protocol for communicating and transferring data effectively across TCP/IP network connections. The mobile application in this system is the HTTP âclientâ. The web server is the HTTP âserverâ. Requests to the flask server are made using HTTP posts and passing objects in JSON format which are parsed on both the client and server. The server will process the data and if requested will return a response to the client. Example from this system is when a âget dataâ request is sent to the server, the server will query database and receive a result set. The server will take the resultset and return it back as a JSON string to the client. Request response messaging

Each route is mapped to a controller, when the controller has a request from a route if successful the controller action will take place.
